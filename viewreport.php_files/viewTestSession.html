Solución a la pregunta número 1
.El objetivo de esta práctica es desarrollar con la ayuda de JFLex y Cup un compilador de un lenguaje denominado PLX generando código intermedio en formato CTD, de acuerdo a las instrucciones que se detallan en este enunciado completo. Para resolver este ejercicio se enviaran todos los ficheros fuentes necesarios para compilar el programa PLXC.class mediante la siguiente secuencia de instrucciones: 
   cup PLXC.cup 
   jflex PLXC.flex
   javac *.java
La compilación deberá llevarse a cabo sin que se produzcan avisos ni conflictos. Una vez compilado el compilador, deberá ejecutarse mediante la instrucción: 
     java PLXC prog.plx prog.ctd
En el fichero plxc-init-minicore.zip, se adjuntan algunos programas de prueba (con extension .plx). Pueden utilizarse versiones compiladas del compilador (plxc). plxc- *.zip para comparar los códigos fuente generados Para ejecutar el código intermedio generado puede usarse el programa ctd 
     ctd prog.ctd
NOTAS:
  • Recuerde que para que funcionen los programas y scripts plxc incluidos en los fichero plxc- *.zip, debe descomprimirlos y modificar los permisos, por ejemplo: 
          unzip plxc-*.zip
      chmod +x plxc
  • Como resultado de este ejercicio deben enviarse TODOS los ficheros necesarios para la compilación del compilador comprimidos en un fichero denominado plxc.zip, (tenga cuidado de no incluir ficheros innecesarios que puedan dificultar la compilación automática). Para obtener el fichero comprimido puede usar la siguiente instrucción: 
         zip plxc.zip PLXC.flex PLXC.cup *.java
  • La corrección de este ejercicio se hace en función de varios casos de prueba independientes, similares, pero NO IGUALES, a los que se incluyen en el fichero comprimido. La correcta compilación de los casos de prueba que se dan en el fichero plxc-init-minicore.zip es condición necesaria, pero no suficiente para superar las pruebas.
